# 安全标准

## 1. 安全目标

- 保护仓库访问
- 保护对象访问
- 保护节点本地缓存与运行目录
- 保护租户和环境隔离
- 保护发布与切换过程的审计完整性

认证、授权、RBAC、mTLS、KMS / Vault 和密钥轮换的详细标准见 `36-认证授权与密钥管理详细设计.md`。

## 2. 认证

- 平台用户认证
- 节点到控制面的机器认证
- Git HTTP 认证
- Git SSH 公钥认证
- S3 访问凭据管理

## 3. 授权

- 平台级角色
- 仓库级权限
- 应用级权限
- 环境级发布权限
- 节点级操作权限
- 运维级故障转移权限

## 4. 隔离

- 多租户数据隔离
- 仓库隔离
- 应用工作目录隔离
- 写回目录与不可变 Release 隔离

## 5. 密钥与凭据

- 凭据不得明文落盘
- S3 密钥支持轮换
- SSH 私钥与机器令牌支持轮换
- 节点本地敏感配置必须最小权限访问

## 6. 审计

必须记录：

- Git push / fetch / mirror 事件
- Release 创建与激活
- 节点切换与回滚
- 配置变更
- 权限变更

## 7. 供应链与内容安全

- Release 对象必须哈希校验
- 可选集成恶意文件扫描
- 正式发布建议支持签名验证

## 8. 运行时安全

- 挂载目录权限最小化
- 同步目录写权限按策略收敛
- 当前版本目录默认只读
- 可写路径集中到 `data/` 与 `workspace/`

## 9. 集群与同步安全

- 节点必须使用机器身份进行注册与通信
- 节点与控制面通信必须支持证书或签名令牌
- Git 镜像与双向同步必须支持远端指纹校验
- 多主模式必须支持操作审计和来源标识

## 10. 威胁模型最低要求

至少需要覆盖以下威胁：

- 非法 Git push
- 恶意对象注入
- 缓存污染
- 节点伪造
- 回滚点篡改
- 双向同步回环
- 节点租约冒用
- 对象墓碑伪造

## 11. 限流与防滥用

- Git push 必须具备速率限制与容量保护
- 发布 API 必须具备幂等与重放保护
- 节点心跳必须具备最小频率限制
- 同步任务必须具备并发上限和隔离队列
