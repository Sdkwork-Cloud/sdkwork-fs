# 测试与验收方案

## 1. 测试目标

- 验证功能完整性
- 验证多语言项目接入能力
- 验证缓存一致性
- 验证切换与回滚可靠性
- 验证集群传播与 Git 同步能力

CI/CD 自动化与灾备演练的详细标准分别见 `40-CI-CD与发布自动化流水线规范.md` 和 `43-故障注入与灾备演练规范.md`。

## 2. 测试层次

### 2.1 单元测试

- Manifest 解析与生成
- 对象索引与哈希校验
- 缓存淘汰策略
- 切换器状态机
- Git 同步策略判断

### 2.2 集成测试

- S3 读写
- Git HTTP / SSH
- 本地 SQLite 状态恢复
- 发布流水线
- 节点预热与切换
- 节点心跳与租约续租
- 写回任务重试与幂等

### 2.3 端到端测试

- React / Node.js 项目发布与切换
- Java Jar 发布与回滚
- Python 包发布与运行
- 大型静态站点同步与切换
- 大文件媒体分块读取
- Git-first 到 Release 到节点运行全流程
- S3-first 到 Release 到节点运行全流程
- 写回冲突副本生成与人工仲裁
- 故障转移后的主写恢复

### 2.4 性能测试

- 冷启动回源
- 热缓存命中
- 多节点并发切换
- 大文件分块读取

### 2.5 故障测试

- 主节点宕机
- 网络分区
- S3 延迟升高
- 缓存损坏
- Git 镜像冲突
- 节点磁盘空间耗尽
- 预热中断
- 双向同步回环配置错误
- 主节点租约丢失
- 节点心跳超时
- 对象墓碑误删保护

## 3. 平台验证矩阵

- Linux
- macOS
- Windows

## 4. 关键验收点

- 发布后节点可成功物化和切换
- 切换失败时可自动回滚
- Git 镜像能按配置单向或双向同步
- 本地目录访问优先于远端回源
- 大文件不进入内存缓存主通道
- 当前版本不会因 GC 被清理

## 5. 多语言验收矩阵

| 类型 | 验收点 |
| --- | --- |
| Node.js / React | 构建产物识别、静态资源预热、目录切换正常 |
| Java | Jar/War 识别、启动文件保留、回滚后可重新启动 |
| Python | wheel/源码目录识别、虚拟环境目录排除、入口脚本可运行 |
| 静态站点 | 索引文件优先预热、切换后页面资源一致 |
| 媒体/大文件 | Range 读取生效、内存缓存不被大文件打爆 |
| 写回/冲突 | 冲突可检测、可保留、可人工介入、不可静默覆盖 |
| 集群/故障转移 | 租约切换后无双主、边缘节点持续可读 |

## 6. 自动化要求

- 单测必须纳入 CI
- 集成测试应覆盖本地对象存储与本地 Git 环境
- 发布回滚链路应有自动回归测试
