# 性能标准

## 1. 核心目标

- 快速读取
- 快速切换
- 可预测的缓存命中率
- 可扩展的集群传播能力

## 2. 指标分类

### 2.1 读取性能

- 本地物化命中：接近本地磁盘性能
- L1 内存命中：近内存访问时间
- L2 对象磁盘命中：显著快于回源 S3
- 冷回源延迟目标应可被监控并随对象大小分段统计

### 2.2 切换性能

- 已预热版本切换：操作本身目标 `<100ms`
- 回滚操作：目标 `<100ms`

### 2.3 发布性能

- Manifest 生成时间
- 缺失对象计算时间
- 节点预热完成时间
- 集群传播延迟

建议初始目标：

- 单应用 Release Manifest 生成小于 `30s`
- 单节点接收发布事件到开始预热小于 `5s`
- 热点对象命中率稳定高于 `80%`
- 关键应用版本传播在同地域内完成于 `60s` 内

## 3. 容量目标

- 单节点可管理多个应用与多个保留版本
- 缓存配额必须可按节点、应用和租户限制
- 中心元数据应支持水平扩容和读写分离演进

### 3.1 容量估算建议

容量估算建议至少考虑：

- `应用数 * 平均保留 Release 数`
- `单 Release 平均对象数`
- `热对象比例`
- `大文件对象占比`
- `双区镜像倍数`

推荐估算公式：

```text
对象磁盘容量 ≈ 应用数 * 保留Release数 * 单Release对象总量 * 去重修正系数
物化目录容量 ≈ 应用数 * 本地保留版本数 * 单版本物化大小
```

其中：

- 去重修正系数通常小于 `1`
- 多语言项目的源码模式与产物模式应分开估算

## 4. 优化策略

- 内容寻址去重
- 并发下载
- 大文件分块与 Range 读取
- 热点文件预热
- 元数据内存缓存
- 按通道和优先级传播发布

### 4.1 压测场景建议

- 小文件高频随机读
- 大文件连续 Range 读取
- 多节点同时预热
- Git 镜像与发布同时进行
- 写回高峰与切换窗口重叠

## 5. 观测指标

- 缓存命中率
- 回源请求数
- 物化耗时
- 切换耗时
- 回滚次数
- Git 同步延迟
- 节点传播延迟

## 6. 性能验收建议

- 小文件密集型应用
- 大文件媒体型应用
- Java Jar 发布
- Python wheel 发布
- 多节点并发切换
- 主节点故障转移期间读服务持续性
- 高并发写回与冲突检测

## 7. 底线要求

- 不允许因缓存 GC 影响当前版本读取
- 不允许因单个大文件拖垮整体内存缓存
- 不允许因一次新发布阻塞当前运行版本
- 不允许因 Git 镜像同步抖动影响当前生产版本访问
