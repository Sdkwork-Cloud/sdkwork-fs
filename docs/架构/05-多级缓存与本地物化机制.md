# 多级缓存与本地物化机制

## 1. 目标

缓存系统必须同时服务以下目标：

- 加速读取
- 降低回源频率
- 支撑快速切换
- 支撑快速回滚
- 支撑离线访问
- 支撑挂载与同步双模式

详细的缓存准入、淘汰、水位、预热和 GC 算法见 `30-缓存策略与淘汰算法规范.md`。

## 2. 缓存层级

### 2.1 L0 元数据内存缓存

内容：

- Manifest
- 路径到对象哈希映射
- Release 指针
- 热门目录树
- 权限与短期 S3 头信息

策略：

- TTL + LRU

### 2.2 L1 热点内容内存缓存

内容：

- 小文件
- 启动脚本
- 核心配置文件
- 热点静态资源

策略：

- 按大小限制
- LRU
- 允许预热

### 2.3 L2 对象磁盘缓存

内容：

- 按哈希去重的对象

建议路径：

```text
objects/sha256/ab/cd/<full-hash>
```

策略：

- 内容寻址
- LRU + 配额
- 多版本共享

### 2.4 L3 本地物化版本缓存

内容：

- 可直接运行的目录结构

建议路径：

```text
apps/<appId>/releases/<releaseId>/
apps/<appId>/current/
apps/<appId>/previous/
```

策略：

- 当前版本固定保留
- 前一个稳定版本默认保留
- 切换前预先物化

### 2.5 L4 工作与写回缓存

内容：

- `data/`
- `queue/`
- `conflicts/`
- `workspace/`

策略：

- 与不可变 Release 隔离
- 支持异步写回
- 支持离线恢复

## 3. 缓存键模型

建议缓存键标准如下：

- 元数据键：`meta:<appId>:<channel>:<releaseId>:<path>`
- 热文件键：`hot:<hash>:<size>`
- 对象键：`obj:<sha256>`
- 物化版本键：`mat:<appId>:<releaseId>`
- 写回任务键：`wb:<appId>:<path>:<op>:<seq>`

## 4. 大文件与媒体缓存

视频、模型、大型二进制不进入 L1。对这类对象采用：

- L2 磁盘缓存
- 分块缓存
- Range 读取
- 热片段预取

视频、音频、模型等媒体型对象建议采用：

- 分块大小可配置
- 最近片段优先保留
- 首片段优先预热
- 全文件下载与流式回源分离统计

## 5. 统一读路径

```text
read(path)
-> 查 L0
-> 查 L3
-> 查 L1
-> 查 L2
-> 回源 S3
-> 校验
-> 写入 L2
-> 物化到 L3
-> 更新 L0 / L1
```

## 6. 统一写路径

```text
write(path)
-> 判断路径是否落入 data/workspace
-> 生成本地写入
-> 记录写回任务
-> 按策略异步或同步上传
-> 更新元数据索引
```

### 6.1 写入策略

- `LOCAL`
- `ASYNC_SYNC`
- `SYNC_STRONG`

默认：

- `release/` 只读
- `data/` 默认 `ASYNC_SYNC`
- `cache/` 默认 `LOCAL`

## 7. 本地物化要求

- 物化目录必须是操作系统可直接访问的真实文件
- Java、Node.js、Python、Nginx 等运行时必须优先读物化目录
- 物化过程不得破坏当前已运行版本

## 8. 缓存回收

支持：

- LRU
- TTL
- 配额控制
- 按版本保留
- Pin 当前版本
- Pin 上一个稳定版本

### 8.1 GC 触发条件

- 磁盘水位超过阈值
- 新 Release 预热前预清理
- 周期性后台清理
- 人工触发

### 8.2 GC 禁止清理对象

- 当前运行版本对象
- previous 回滚版本对象
- 正在预热的目标版本对象
- 进行中写回任务依赖对象

## 9. 预热机制

支持三类预热：

- 发布预热
- 切换前预热
- 热点文件自动预热

### 9.1 预热优先级

1. 启动入口文件
2. 配置文件
3. 首页和索引文件
4. 热点静态资源
5. 其余对象

## 10. 一致性要求

- Manifest 与对象哈希不一致时必须阻止激活
- 本地缓存损坏时必须允许自动重拉
- 任何 GC 不得删除当前运行版本依赖的对象
