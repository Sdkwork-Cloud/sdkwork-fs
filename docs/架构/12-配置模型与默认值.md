# 配置模型与默认值

## 1. 设计目标

- 高度可配置
- 有明确默认值
- 支持多级覆盖
- 支持热更新与重载边界

完整可复用模板见 `32-配置示例与环境模板.md`。

## 2. 配置层级

按优先级从低到高：

1. 平台默认
2. 仓库覆盖
3. 应用覆盖
4. 环境覆盖
5. 节点覆盖

## 3. 标准配置项

### 3.1 Git 工作流

```yaml
git:
  workflow:
    mode: trunk-based
    mainBranch: main
    devSource: main
    testSource: main
    prodSource: tag
```

### 3.2 Git 同步

```yaml
git:
  replication:
    mode: mirror
    enabled: true
    branchFilter:
      - main
      - release/*
    tagFilter:
      - v*
    loopPrevention: true
    conflictPolicy: reject
```

### 3.2.1 受管裸仓库存储

```yaml
sdkworkfs:
  git:
    managed:
      rootDir: ./var/sdkworkfs/git/repos
```

说明：

- `sdkworkfs.git.managed.root-dir` 用于声明控制面受管 bare repo 的根目录
- Git 同步成功后的仓库内容应落到 `<rootDir>/<tenantId>/<repositoryId>.git`
- 控制面应把受管 bare repo 绝对路径回写到仓库投影字段，供 build、mirror、诊断复用
- `build-release` 应优先消费本地 bare repo，只有策略允许时才回退到上游远端

### 3.2.2 Git 候选 Release 产物选择

```yaml
repository:
  source:
    buildProfile: react
    buildOutput: dist/
```

说明：

- `buildOutput` 为显式产物选择规则，优先级高于 `runtimeType` 默认规则
- 平台应同时支持仓库级与应用级 `buildOutput` 覆盖
- `buildOutput` 只定义产物路径选择，不等价于任意构建命令注入

### 3.2.3 平台级真实构建执行器

```yaml
sdkworkfs:
  build:
    workspaceRootDir: ./var/sdkworkfs/build/workspaces
    commandPathPrepend: /opt/build-tools/bin
    node:
      enabled: false
      installTimeoutSeconds: 300
      buildTimeoutSeconds: 300
    java:
      enabled: false
      mavenTimeoutSeconds: 600
      gradleTimeoutSeconds: 600
    python:
      enabled: false
      buildTimeoutSeconds: 600
```

说明：

- 平台通过 `sdkworkfs.build.*` 暴露构建工作区、命令 PATH 前置，以及 Node.js / React、Java、Python 的受控构建开关
- `node.enabled`、`java.enabled` 与 `python.enabled` 默认关闭，避免在未准备好工具链的环境中误执行真实构建
- `commandPathPrepend` 用于把 `pnpm` / `npm` / `yarn` / `mvn` / `gradle` / `python` 所在目录前置到 `PATH`，便于 Windows、自定义安装路径或虚拟环境下稳定解析命令
- `java.mavenTimeoutSeconds` 与 `java.gradleTimeoutSeconds` 分别控制 Maven / Gradle 固定模板构建超时
- `python.buildTimeoutSeconds` 控制固定模板 `python -m build --wheel --sdist` 的执行超时，并要求目标解释器预装 `build` 模块
- 默认交付镜像不内置 Node.js、Maven、Gradle 或 Python build toolchain；只有控制面宿主机、自定义镜像或虚拟环境已提供工具链时才建议开启

## 4. 默认值原则

- 默认值优先稳定性
- 高风险能力默认关闭
- 受管 bare repo 本地落盘默认开启
- 真实语言级构建默认关闭
- 双向 Git 同步默认关闭
- 多主集群默认关闭
- 生产正式发布默认使用 tag/release
- 对象上传完成校验默认开启
- 本地开发默认允许控制面自动创建对象存储 bucket
- 生产环境建议保持 missing bucket fail-fast

## 5. 热更新边界

可热更新：

- 缓存配额
- 预热策略
- Git 同步过滤规则
- 节点拉取频率

需重启或重建会话：

- 挂载点
- 本地根目录
- 核心存储连接模式

## 6. 配置校验

- 启动前校验
- 变更时校验
- 提供 dry-run 校验能力
- 错误配置不得静默生效
