# SLO 与告警规则细化规范

## 1. 目标

在观测与运维 Runbook 基础上，把 SLI、SLO、告警分级、Prometheus 规则和通知矩阵细化为可配置、可实施、可验收的标准。

## 2. 核心 SLI

至少定义：

- Release 创建成功率
- Release 激活成功率
- 节点心跳可用率
- 切换耗时
- 回滚耗时
- Git 镜像成功率
- S3 回源错误率
- 缓存命中率

## 3. 推荐 SLO

| SLI | SLO |
| --- | --- |
| Release 创建成功率 | `>= 99.9%` |
| Release 激活成功率 | `>= 99.95%` |
| 已预热切换耗时 P95 | `< 500ms` |
| 节点心跳可用率 | `>= 99.99%` |
| Git 镜像任务成功率 | `>= 99.5%` |

## 4. 告警分级

- `P1`：生产发布或切换核心能力不可用
- `P2`：关键能力退化，但存在绕行路径
- `P3`：容量、趋势、风险预警

## 5. 典型告警规则

### 5.1 节点心跳超时

触发条件：

- 连续多个周期未收到心跳

建议规则表达：

```text
count(node_heartbeat_timeout_total{environment="prod"}) > 0
```

### 5.2 缓存磁盘高水位

触发条件：

- 使用率超过 `85%`

### 5.3 回源错误率升高

触发条件：

- 指定时间窗内对象回源错误率持续超阈值

### 5.4 金丝雀失败

触发条件：

- 错误率、延迟、回滚数任一超阈值

## 6. 通知矩阵

| 级别 | 通知对象 |
| --- | --- |
| `P1` | 值班、平台负责人、运维负责人 |
| `P2` | 值班、模块负责人 |
| `P3` | 模块负责人、周报聚合 |

## 7. 抑制与降噪

- 已知维护窗口内允许抑制非关键告警
- 同一事故根因产生的重复告警应聚合
- 频繁抖动指标需增加持续时间条件

## 8. 仪表盘最低要求

- 发布链路总览
- 节点健康总览
- 缓存与回源总览
- Git 同步总览
- 回滚与金丝雀总览

## 9. 正式要求

- 每个关键 SLO 必须绑定至少一个可计算 SLI
- 每个 P1/P2 告警必须映射 Runbook
- 告警、仪表盘和通知矩阵必须作为生产标准配置的一部分
